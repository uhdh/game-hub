# 블라인드 경매 (Blind Auction) — 설계 문서

## 개요
싱글플레이 경매 게임. 유저 1명 + AI 4명이 큐브(가상 화폐)로 11장의 물품 카드를 순서대로 경매한다(플레이어당 큐브 11개와 대응하는 11라운드제). 각 물품의 실제 가치는 낙찰 전까지 비공개이며, 11장의 총합만 게임 시작 시 공개된다. 게임 종료 시 낙찰받은 물품 가치 합계가 가장 높은 사람이 승리한다.

기존 프로젝트(모자이크퍼즐 게임 허브) 컨벤션을 그대로 따른다: 순수 HTML/CSS/JS 단일 파일, 다크 테마, `index.html` GAMES 배열에 카드 추가, Supabase(`paktzmofotvwfdxcpmzv` 프로젝트) REST API를 raw fetch로 사용.

## 파일 구성
- `blind-auction.html` — 게임 본체 (신규)
- `admin.html` — 기존 파일에 "경매 물품 관리" 탭 추가 (기존 "탈락자 관리" 탭과 공존)
- `index.html` — GAMES 배열에 새 카드 추가 (아이콘: 🔨 이모지)

## 데이터 모델 (Supabase)

### `auction_items` 테이블
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | uuid/serial (PK) | |
| item_name | text | 물품명 |
| value | integer | 실제 가치 (관리자가 입력, 게임 중엔 비공개) |
| memo | text (nullable) | 설명/메모 — 낙찰 시 함께 공개되는 플레이버 텍스트 |
| created_at | timestamptz | |

RLS: anon 전체 개방 (기존 테이블들과 동일 패턴 — 공개 정적 사이트, anon publishable key만 노출).

### `blind_auction_leaderboard` 테이블
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | uuid/serial (PK) | |
| nickname | text | |
| score | integer | 유저가 낙찰받은 물품 가치 합계 (게임 종료 시점) |

`rearrange_leaderboard`와 동일하게 매 게임 1건 insert, 상위 10건을 `score.desc, created_at.asc`로 조회.

## 관리자 페이지 (admin.html 확장)

기존 비밀번호 게이트(`ADMIN_PASSWORD`) 재사용. 새 탭 "경매 물품 관리" 추가:

- **현황 표시**: 전체 등록 건수. 11건 미만이면 "게임 시작 불가 — N건 더 필요" 경고 표시.
- **단건 입력 폼**: 물품명(필수) / 가치(필수, 정수) / 메모(선택) 입력 후 등록.
- **일괄 붙여넣기**: 텍스트박스에 한 줄당 `물품명, 가치, 메모` (메모는 생략 가능) 형식으로 여러 줄 붙여넣으면 파싱해서 일괄 insert. 파싱 실패한 줄은 건너뛰고 결과 요약(성공 N건 / 실패 N건)을 보여줌.
- **목록**: 최신순으로 나열, 각 항목에 삭제(×) 버튼. 수정 기능은 없음 — 삭제 후 재등록.

## 게임 진행 로직

### 준비 단계
1. `auction_items` 전체를 조회 (개수 < 11이면 시작 화면에서 차단, 관리자 페이지로 안내).
2. 클라이언트에서 셔플 후 11개를 추출.
3. 11개 가치의 합계를 계산해 화면에 공개.
4. 유저 + AI 4명에게 각각 큐브 세트 `{1,2,3,4,6,8,10,12,15,20,25}` (11개, 합계 106) 지급.
5. 추출된 11개 아이템을 다시 셔플해 이번 게임의 "등장 순서 큐"로 확정.

### 라운드 (물품 1장 경매)
- **선 플레이어 결정**: 첫 라운드는 랜덤. 이후 라운드는 직전 라운드의 낙찰자. 직전 라운드가 유찰이었던 경우, 그 전 선 플레이어의 다음 순번(고정 로테이션: 유저→AI1→AI2→AI3→AI4→유저…)으로 넘어간다(같은 사람이 계속 선을 독점하지 않도록).
- **턴 진행**: 아직 패스하지 않은("활성") 플레이어를 고정 순서로 순회.
  - 현재 최고 입찰자인 플레이어는 자기 차례가 와도 스킵(이미 이기고 있으므로 행동 불필요).
  - 그 외 활성 플레이어는 자기 차례에 다음 중 하나를 선택:
    - **입찰**: 아직 이번 경매에 쓰지 않은 보유 큐브 중 일부를 추가로 선택해, "이번 경매에서 내가 누적 제출한 합계"가 현재 최고가를 초과하도록 만든다. 한 번 제출한 큐브는 이번 경매가 끝날 때까지(포기해도) 회수 불가 — 계속 쌓이기만 한다.
    - **포기**: 이번 경매에서 완전히 이탈한다(재참여 불가). 제출했던 큐브는 경매가 끝날 때까지 묶여있다가, 경매 종료 시 일괄 반환된다.
  - 활성 플레이어가 1명만 남으면 그 사람이 낙찰. 즉시 값/메모 공개, 낙찰자는 제출한 큐브를 영구 소모, 나머지 플레이어는 이번 경매에 묶어뒀던 큐브를 전원 반환받는다.
  - 아무도 입찰하지 않고(선 플레이어부터 전원 0원 상태로 패스) 전원 이탈하면 **유찰** 처리: 값/메모는 공개하되 아무도 획득하지 못하고 다음 라운드로 넘어간다.

### 종료
- 11라운드(등장 순서 큐 소진) 종료 후, 각 플레이어가 낙찰받은 물품 가치의 합계로 최종 순위를 매긴다. 남은 큐브는 점수에 반영하지 않는다.
- 결과 화면에 참가자별(유저+AI4) 낙찰 물품 목록과 합계를 표시.
- 유저는 닉네임을 입력해 자신의 합계를 `blind_auction_leaderboard`에 등록할 수 있다.

## AI 입찰 로직

- 게임 시작 시 AI 4명 각각에게 "적극성 계수"를 0.8~1.3 범위에서 랜덤 배정(매 게임 성향이 달라짐).
- 라운드별로 AI의 "의향가" = (공개된 11개 평균가 = 총합 ÷ 11) × 적극성 계수 × 소폭의 라운드별 랜덤 편차. 남은 큐브 총액과 남은 라운드 수를 고려해 상한을 둬서 초반 몰빵을 방지한다(예: 의향가는 "남은 큐브 총액 ÷ 남은 라운드 수"를 크게 벗어나지 않도록 클램프).
- 자기 차례에: 현재 최고가가 의향가보다 낮고, 이를 넘길 수 있는 큐브 조합이 남아있으면 "최고가를 막 초과하는 최소 조합"을 그리디하게 선택해 입찰. 그렇지 않으면 포기.
- 유저는 AI의 남은 큐브 총액을 볼 수 없다(자신의 큐브만 공개). AI의 실제 판단 로직에는 영향 없음 — UI 비공개일 뿐.

## UI 화면 구성

기존 게임들과 동일하게 다크 테마(`#0f0f12` 배경, `#15181c` 카드), 단일 페이지 내 상태 전환 방식.

1. **시작 화면**: 룰 요약 텍스트 + 등록된 물품 수 체크(11개 미만 시 시작 차단 안내) + "게임 시작" 버튼.
2. **합계 공개 화면**: "11개 물품 가치 합계: N" 표시 → "경매 시작" 버튼.
3. **경매 화면(메인)**: 현재 물품명(값은 "?"), 현재 최고가/최고 입찰자, 현재 차례 표시, 5인 플레이어 상태 바(닉네임, 활성/패스 상태 뱃지, 유저만 보유 큐브 합계 노출), 유저 큐브 트레이(11개 토글 버튼, 이미 쓴 큐브는 비활성 표시), "입찰"/"포기" 버튼, 진행 로그(누가 얼마를 냈는지 텍스트 리스트).
4. **낙찰/유찰 모달**: 물품명·값·메모·낙찰자(또는 "유찰") 공개 → "다음 물품" 버튼.
5. **결과 화면**: 최종 순위표(참가자별 낙찰 물품 목록 + 합계), 승자 하이라이트, 닉네임 입력 후 리더보드 등록, 상위 10위 리더보드 목록, "다시하기"/"홈으로" 버튼.

허브 홈(`index.html`)에는 새 게임 카드 추가(🔨 이모지 아이콘, `./blind-auction.html`, target `_self`).
