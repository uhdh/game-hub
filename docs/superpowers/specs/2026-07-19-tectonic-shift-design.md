# 지각변동3인 (Tectonic Shift) — 설계 문서

## 개요
포인티톱 육각 타일 보드 위에서 최대 3명(사람/AI 혼합)이 번갈아 자기 말을 직선으로 밀어 이동시키며 점수를 쌓는 로컬 핫시트 보드 게임. 디자인 핸드오프(`기존 웹 디자인안 3개_지각변동.zip` → `design_handoff_game_hub/Jigak Byeondong 3P.dc.html` + `Home.dc.html`)의 게임 로직·좌표계·시각 스펙을 그대로 이식하되, AI 대전 모드는 이번 설계에서 새로 추가한다.

기존 프로젝트 컨벤션을 그대로 따른다: 순수 HTML/CSS/JS 단일 파일, 다크 테마, `index.html` GAMES 배열에 카드 추가, Supabase(`paktzmofotvwfdxcpmzv` 프로젝트) REST API를 raw fetch로 사용(SDK 미사용), 랜덤 닉네임 자동 배정 패턴 재사용.

## 파일 구성
- `tectonic-shift.html` — 게임 본체 (신규)
- `index.html` — GAMES 배열에 새 카드 추가, "3단 오목" 바로 뒤에 삽입

## 게임 로직 (디자인 원본 그대로 이식)

### 보드 정의
- `ROW_DEFS`: 13개 행, 폭 7→13→7칸(대칭), 각 칸에 점수(0~5) 또는 `null`(시작부터 제거된 칸) 지정. 원본 파일의 배열 값을 그대로 사용:
  ```
  행0:  [1,0,0,0,0,0,1]                              pieces: 1~5=G(초록)
  행1:  [1,2,2,2,2,2,2,1]
  행2:  [1,2,3,3,3,3,3,2,1]
  행3:  [1,2,3,4,4,4,4,3,2,1]
  행4:  [1,2,3,4,1,1,1,4,3,2,1]
  행5:  [1,2,3,4,5,null,null,5,4,3,2,1]
  행6:  [1,2,3,4,1,null,null,null,1,4,3,2,1]
  행7:  [0,2,3,4,1,null,null,1,4,3,2,0]               pieces: 0=R(빨강), 11=Y(노랑)
  행8:  [0,2,3,4,1,5,1,4,3,2,0]                        pieces: 0=R, 10=Y
  행9:  [0,2,3,4,4,4,4,3,2,0]                          pieces: 0=R, 9=Y
  행10: [0,2,3,3,3,3,3,2,0]                            pieces: 0=R, 8=Y
  행11: [0,2,2,2,2,2,2,0]                              pieces: 0=R, 7=Y
  행12: [1,1,1,1,1,1,1]
  ```
  총 127칸, 그중 7칸(행5~7의 null)은 시작부터 제거된 통행 불가 칸. 각 색상당 시작말 5개.
- 좌표계: 표준 axial/cube 육각좌표 대신, 각 타일의 픽셀 좌표(`x, y`)를 반올림한 문자열을 키로 하는 해시맵(`posKey`)에 저장하고 6개의 고정 방향 벡터(`DIRS: {dx,dy}`)를 더해 이웃 칸을 조회. 행마다 폭이 다른 비정형 육각형이라 이 방식을 그대로 유지(원본과 동일 이유 — 표준 공식 적용 어려움).

### 이동 규칙
- 자기 차례에 자신의 말 1개를 6방향 중 하나로 선택 — 그 방향으로 갈 수 있는 데까지(가장자리·제거된 칸·다른 말에 막히기 전까지) **끝까지** 직선 이동(중간 정지 불가, 도달 가능한 마지막 칸에만 착수 가능).
- 이동 후: 출발 칸의 점수를 획득 + 출발 칸은 영구 제거(통행 불가 처리). 도착 칸으로 말 이동.
- 이동 가능한 칸이 하나도 없는 말은 선택 자체가 불가.

### 고립 자동 제거 (`sweepIsolation`) — 구현 중 발견한 원본 설계의 빈틈 보완
매 이동 후 모든 말에 대해 반복 판정: 인접 6칸이 전부 "제거됨" 상태라 이동할 곳이 전혀 없는 말은 점수 없이 즉시 보드에서 제거. 단, 이동 불가 사유 중 하나라도 "인접 칸에 상대 말이 있어서"인 경우는 제거하지 않고 보류(그 상대 말이 나중에 비켜나면 재판정).

**원본 프로토타입 그대로 이식했을 때 발견된 문제**: 원본은 말 하나를 단독으로만 판정한다. 그런데 서로 다른 색 말 여러 개가 "서로만" 막고 있는 닫힌 덩어리(예: A는 B에게만 막히고 B는 A에게만 막혀 있고, 그 그룹 전체가 빈 칸으로 나가는 길이 하나도 없는 경우)는 한 칸씩 따로 보면 항상 "상대 말에 막힘"으로만 보여 원본 로직으로는 영원히 제거되지 않는다 — 실제 플레이 테스트에서 3인 전원이 동시에 이동 불가 상태로 고착되는 상황이 재현됨. 이런 그룹은 실제로는 전원이 절대 못 움직이는 죽은 그룹이므로, 인접한 말들을 그래프로 묶어 **그룹 전체에 빈 칸으로 나가는 경로가 단 하나도 없으면 그룹째 동시에 고립 제거**하도록 고쳤다(원본 개념의 확장 — 낱개 말 판정을 "말 1개짜리 그룹"으로 보면 원본 규칙과 동일하게 동작).

**추가 안전장치**: 위 수정으로도 이론상 발생 가능한 전원 교착 상태에 대비해, 생존한 모든 플레이어가 동시에 이동 불가면(턴 넘기기를 반복해도 아무도 이동 못 함) 누적 점수(동점이면 잔여 말 수) 비교로 즉시 종료시키는 보조 로직(`allPlayersStuck`/`resolveDeadlock`)도 추가했다.

### 턴 진행
- 순서: 빨강(R) → 초록(G) → 노랑(Y) 고정 로테이션, 제거된(전멸) 플레이어는 건너뜀.
- 90초 타이머는 **화면 표시만** 하고 자동으로 턴을 넘기지 않는다(원본 스펙 그대로 — 시간 압박용 표시 요소일 뿐 강제성 없음). 매 이동 시 90초로 리셋.
- "턴 넘기기" 버튼은 현재 차례 플레이어가 이동 가능한 말이 하나도 없을 때만 노출.

### 종료·승리
한 플레이어의 말 5개가 모두 제거되는 즉시 종료. 전멸한 플레이어를 포함해 3인 전원 중 누적 점수 최다 플레이어 승리(전멸했다는 이유만으로 후보에서 제외하지 않음), 동점이면 보드에 남은 말이 더 많은 쪽 승리. 승리 모달에 사유 텍스트와 함께 표시.

## AI 모드 (신규 설계 — 원본 프로토타입엔 AI 없음)

### 모드 프리셋 (2개, 고정)
1. **3인 로컬 플레이**: R/G/Y 전원 사람 (핫시트).
2. **1인 vs AI 2명**: 사람 = 빨강(R) 고정, AI = 초록(G) · 노랑(Y). (색상 선택 UI는 만들지 않음 — gomoku-stack.html의 "AI와 대결"과 동일하게 고정 배정으로 단순화)

### AI 턴 처리
AI 차례가 되면 상태 표시를 "○○ AI가 생각하는 중..."으로 바꾸고 0.5~1초 지연 후 자동으로 이동 실행. 사람의 버튼 조작 불필요(오목의 AI vs AI 관전 모드처럼 "재생" 버튼 없이 자동 진행).

### AI 의사결정 로직 (단일 난이도, 이 게임 전용 신규 휴리스틱)
1. 자기 말들 전부에 대해 `legalDestinations`로 합법 이동 후보(origin→dest) 전부 나열.
2. 각 후보를 시뮬레이션(이동 적용 + `sweepIsolation` 실행)해서 다음 기준으로 점수 계산:
   - **즉시 획득 점수** (origin 칸의 점수) — 가장 큰 가중치.
   - **자기 말 추가 고립 손실**: 이 수를 둔 결과 시뮬레이션에서 자기 말이 (이 이동 자체가 아니라) 고립으로 추가 제거되면 큰 감점.
   - **기동성 우위**: 이동 후 내 남은 말들의 총 이동 가능 칸 수 − 상대 말들의 총 이동 가능 칸 수. 약한 가중치로 반영(즉시 점수보다 후순위).
3. 최고 점수 후보를 선택, 동점이면 무작위로 하나 선택.
4. 이동 가능한 말이 하나도 없으면(전원 고립 등 이례적 상황) AI 쪽도 "턴 넘기기"와 동일한 처리로 자동 패스.

## 닉네임 & Supabase 기록

### 새 게임 시작 시
- **3인 로컬 플레이**: 빨강·초록·노랑 3개 닉네임 입력란 노출. 비워두면 기존 랜덤 닉네임 규칙(`이름#숫자`, 오목/재배치/경매와 동일한 이름 풀 재사용) 자동 배정. 각 색상의 마지막 입력값은 localStorage(`tectonicShiftNickR/G/Y`)에 저장해 다음 판에 미리 채움.
- **1인 vs AI 2명**: 사람(빨강) 닉네임 입력란 1개만 노출(빈 값이면 랜덤 배정 + localStorage 기억). 초록·노랑은 AI 고정이므로 닉네임 없음.

### `tectonic_shift_results` 테이블 (신규)
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | uuid/serial (PK) | |
| created_at | timestamptz | |
| mode | text | `'3p'` 또는 `'vs_ai'` |
| nicknames | jsonb | `{R,G,Y}` — 사람 자리만 닉네임 문자열, AI 자리는 `null` |
| scores | jsonb | `{R,G,Y}` 최종 누적 점수 |
| winner_color | text | `'R'`/`'G'`/`'Y'` |
| winner_nickname | text (nullable) | 승자가 AI면 `null` |
| win_reason | text | 종료 사유 텍스트(원본 문구 그대로) |

RLS: anon insert + select만 허용(다른 게임 로그 테이블과 동일 패턴, update/delete 없음).

### 승리 랭킹 패널
`winner_nickname is not null`인 행만 집계 대상(AI 승리는 자연스럽게 제외). 클라이언트에서 raw fetch로 전체 행을 가져와 `winner_nickname` 기준 그룹핑 후 승수 내림차순 상위 10명 표시(predict.html의 투표 집계와 동일한 클라이언트 집계 방식 — 별도 서버 집계 불필요).

## 보드 시각화
- 타일 렌더링: CSS `clip-path` 대신 각 타일 안에 `<svg><polygon></svg>`를 겹쳐 그리는 방식 그대로 사용(원본 사유: `clip-path`는 포인티톱 육각형의 위/아래 꼭짓점 테두리가 그려지지 않는 문제가 있음). `stroke:#000;stroke-width:1`.
- 배경(빈 타일) 색상: `#f0ece2` (3단 오목 흰돌 색과 통일).
- 점수 텍스트 색: 0=`#9aa1ab`, 1=`#8e6fd1`, 2=`#4aa3e0`, 3=`#3f9b6f`, 4=`#dba23c`, 5=`#e0524a`.
- 말 색상: 초록 `#7cbb45`, 빨강 `#e0453f`, 노랑 `#f2c53c`, 원형 26px, 테두리 `2px solid rgba(0,0,0,.25)`.
- 선택한 말 → 이동 가능한 각 목적지까지 실선(`#5fb8b0`, 3px) 연결, 목적지 타일 배경은 민트(`#8fd9cf`)로 강조.
- 마지막으로 이동한 말의 경로는 점선(플레이어 색, 4px, dasharray) + 출발점 원으로 표시, 다음 이동 전까지 유지.
- SVG(폴리곤/선) 레이어는 `position:absolute`라 별도 stacking context가 생기므로, 그 위에 그려야 하는 숫자/말 요소들에는 `position:relative;z-index:1` 명시 필요(원본에서 명시한 주의사항, 재구현 시에도 유지).

## UI 화면 구성
1. **시작 화면**: 모드 선택(3인 로컬 플레이 / 1인 vs AI 2명) → 선택한 모드에 맞는 닉네임 입력란 → "게임 시작" 버튼.
2. **게임 화면(메인)**: 상단 "← 게임 허브" 링크 + 타이틀 + "새 게임" 버튼. 플레이어 칩 3개(닉네임/AI 표시, 점수, 남은 말 수, 현재 차례 강조). 상태 바(현재 차례, 90초 타이머 표시, 조건부 "턴 넘기기" 버튼). 육각 보드. 규칙 요약 박스.
3. **승리 모달**: 승자 색/닉네임(또는 "AI") + 승리 사유 + "새 게임 시작" 버튼. (AI 모드에서 사람이 이겼을 때만 결과가 랭킹에 반영됨을 별도로 알리지는 않음 — 조용히 처리)
4. **승리 랭킹 패널**: 게임 화면 하단에 상시 노출, 상위 10명(닉네임 + 승수).

허브 홈(`index.html`)에는 새 게임 카드 추가:
- 이름: "지각변동3인", desc: "무너지는 육각 타일 위에서 점수 높은 땅으로 말을 밀어붙이세요"(디자인 원본 그대로)
- 위치: GAMES 배열에서 "3단 오목" 바로 뒤
- 아이콘: 신규 `hexIcon` 타입 추가(기존 `quads`/`image`/`emoji` 3종에 추가) — 56×56 컨테이너 안에 32×37 포인티톱 육각형 SVG 4개를 `position:absolute`로 겹쳐 배치. 디자인 원본(`Home.dc.html`)의 정확한 값:
  1. `left:14px; top:-9px`, 배경 `#f0ece2`, 라벨 `5`, 라벨색 `#c97a76`
  2. `left:28px; top:20px`, 배경 `#f0ece2`, 라벨 `1`, 라벨색 `#a89bc9`
  3. `left:0; top:20px`, 배경 `#f0ece2`, 라벨 `4`, 라벨색 `#c9a56b`
  4. `left:-14px; top:-6px`, 배경 `#141414`(검정, 라벨 없음)
  각 항목의 폴리곤 포인트는 게임 보드와 동일한 `16,0 32,9.25 32,27.75 16,37 0,27.75 0,9.25`.

## 범위 밖 (Out of scope)
- AI 난이도 다단계(하/중/상) — 단일 난이도만 지원, 추후 필요 시 별도 작업.
- AI vs AI 관전 모드 — 이번엔 "1인 vs AI 2명"까지만, 전원 AI 자동진행 모드는 만들지 않음.
- 플레이어가 어떤 색을 맡을지 선택하는 UI — 사람은 항상 빨강(R) 고정.
- 리플레이/기보 저장 — 승패 결과 1건만 기록, 오목처럼 수순(moves) 로그는 남기지 않음.
- 온라인 실시간 대전 — 로컬 핫시트 + 로컬 vs AI만 지원.
